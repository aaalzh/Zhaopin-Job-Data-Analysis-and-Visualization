# 智联招聘爬虫数据分析系统企业级开发步骤文档

## 1. 文档说明

### 1.1 文档目的

本文档根据《智联招聘爬虫数据分析需求文档》和《智联招聘爬虫数据分析系统详细设计文档》编写，用于指导四人团队在 2 周内完成系统开发、联调、测试、文档整理和答辩准备。

本文档重点解决以下问题：

1. 明确四名成员的职责边界。
2. 明确每个阶段的输入、输出和交接间隔。
3. 明确开发顺序，避免多人同时修改同一模块造成冲突。
4. 明确每日检查点、验收标准和风险处理方式。

### 1.2 项目周期

项目周期为 2 周，建议按 10 个工作日执行，周末作为缓冲、补测和答辩材料整理时间。

| 周期 | 时间范围 | 目标 |
| --- | --- | --- |
| 第 1 周 | 第 1 天至第 7 天 | 完成环境、爬虫、清洗、HDFS、Hive 基础开发 |
| 第 2 周 | 第 8 天至第 14 天 | 完成 MySQL、可视化、联调、测试、文档和答辩材料 |

如从 2026-06-15 开始，可按 2026-06-15 至 2026-06-28 执行；如实际开工日期不同，整体顺延即可。

### 1.3 系统开发范围

本项目采用离线数据分析架构，不单独开发 Java 后端。开发范围包括：

1. Python 爬虫采集模块。
2. Redis 任务队列、去重、失败重试模块。
3. Python 数据清洗、字段标准化和文件导出模块。
4. HDFS 数据上传和目录管理模块。
5. Hive ODS、DWD、DWS、ADS 数仓分层和统计 SQL。
6. MySQL 任务日志表和分析结果表。
7. Python Hive 到 MySQL 同步、结果查询和 JSON 导出脚本。
8. ECharts 静态可视化页面。
9. 测试报告、部署说明、项目总结和答辩材料。

3. 运行证据：命令、日志、截图或查询结果。
4. 问题清单：未解决异常、临时方案和下一步建议。

### 7.2 数据验收标准

1. 至少完成 1 个岗位关键词、1 个城市、5 页数据采集。
2. 有效岗位数据不少于 100 条。
3. `clean.csv` 字段完整，核心字段缺失率控制在合理范围内。
4. Hive 中能查询到清洗后的岗位明细数据。
5. MySQL 中能查询到分析结果数据。
6. ECharts 页面读取的 JSON 与 MySQL 结果一致。

### 7.3 代码质量标准

1. 配置参数不得硬编码在多个文件中，统一放入配置文件。
2. 爬虫请求必须设置请求间隔、超时和最大重试次数。
3. 原始 JSON 必须保留，便于字段变化后重新解析。
4. 清洗逻辑必须独立封装，不能写死在主程序中。
5. Hive SQL 按 ODS、DWD、DWS、ADS 分目录管理。
6. MySQL 同步脚本必须支持重复执行，避免重复写入。
7. ECharts 图表数据字段必须有统一 JSON 格式说明。
8. 每个模块必须有最小运行说明和测试证据。

## 8. 风险与应对方案

| 风险 | 影响 | 负责人 | 应对方案 |
| --- | --- | --- | --- |
| 页面出现安全验证 | 采集失败 | 黎 | 降低请求频率，记录异常，不绕过验证，优先使用公开 JSON 数据 |
| 接口字段变化 | 解析失败 | 黎、杨 | 保留原始 JSON，字段映射集中维护 |

| MySQL 同步重复写入 | 结果翻倍 | 秦 | 同步前按 `crawl_date + keyword` 删除旧数据 |
| JSON 字段不一致 | 图表无法渲染 | 邓 | 固定 JSON 格式，前端增加空数据处理 |
| 文档与代码不一致 | 答辩扣分 | 邓主责，全员 | 第 13 天统一文档评审，所有截图重新核对 |
